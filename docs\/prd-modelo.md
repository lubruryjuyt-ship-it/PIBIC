#Modelo de PRD - ORIENTAÇÕES

Projeto – Desenvolvimento de Website

1. O que é o PRD?
PRD significa Product Requirements Document (Documento de Requisitos do Produto).
Trata-se de um documento técnico que descreve, de forma sistemática e detalhada:

• o que será desenvolvido
• por que será desenvolvido
• para quem será desenvolvido
• como deverá funcionar

O PRD antecede a fase de implementação (HTML, CSS e JavaScript). Nenhum código deve ser iniciado antes da definição clara dos requisitos.

────────────────────────

2. Estrutura obrigatória do PRD
O documento deve conter as seguintes seções:

2.1. Identificação do Projeto
• Nome do projeto
• Nome dos orientandos
• Nome da orientadora: profa. Dra. Nívia Maria Assunção Costa
• Data
• Versão do documento

────────────────────────

2.2. Contextualização e Justificativa
Responder:
• Qual problema este website busca resolver?
• Qual sua relevância acadêmica ou social?
• Qual público será impactado?
A justificativa deve apresentar clareza, coerência e fundamentação lógica.

────────────────────────

2.3. Objetivo Geral
Descrever em um único parágrafo o propósito central do website.

Exemplo de estrutura:
Desenvolver um website interativo com a finalidade de ________, voltado para ________, utilizando tecnologias front-end (HTML, CSS e JavaScript).

────────────────────────

2.4. Público-Alvo
Definir:

• faixa etária
• perfil do usuário
• nível de conhecimento digital
• necessidades principais

────────────────────────

2.5. Escopo do Projeto
Descrever o que será incluído e o que não será incluído.

Exemplo:

Inclui:
• Página inicial
• Página informativa
• Formulário interativo
• Elementos visuais responsivos

Não inclui:
• Banco de dados
• Backend
• Integração com APIs externas

────────────────────────

2.6. Requisitos Funcionais
Descrevem o que o sistema deve fazer.

Exemplos:
• O sistema deve permitir navegação entre páginas.
• O sistema deve validar campos obrigatórios no formulário.
• O sistema deve exibir mensagens de erro quando necessário.

Usar sempre a estrutura:
O sistema deve...

────────────────────────

2.7. Requisitos Não Funcionais
Descrevem características de qualidade.

Exemplos:
• O site deve ser responsivo.
• O carregamento deve ocorrer em até 3 segundos.
• O código deve seguir boas práticas de organização.
• O layout deve ser acessível.

────────────────────────

2.8. Estrutura do Site (Arquitetura)
Apresentar:

• Mapa de navegação
• Estrutura de pastas
• Organização dos arquivos

Pode incluir um pequeno diagrama ou lista hierárquica.

────────────────────────

2.9. Tecnologias Utilizadas
• HTML5
• CSS3
• JavaScript

Descrever brevemente a função de cada tecnologia no projeto.

────────────────────────

2.10. Critérios de Sucesso
Definir como será avaliado se o projeto foi bem-sucedido.

Exemplo:
• Todos os requisitos funcionais implementados
• Navegação sem erros
• Estrutura organizada conforme padrão definido
• Código comentado adequadamente

────────────────────────

4. Orientações Metodológicas
• O PRD deve ser redigido antes da implementação.
• Deve ser claro, objetivo e tecnicamente fundamentado.
• Evitar descrições vagas.
• Utilizar linguagem formal.
• Versões devem ser registradas no repositório.

────────────────────────

6. Avaliação
O PRD será avaliado considerando:
• Coerência interna
• Clareza dos requisitos
• Organização estrutural
• Alinhamento com o objetivo do projeto
• Viabilidade técnica
